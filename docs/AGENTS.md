# docs/ — plans and working documents

- **`CodebaseReview-2026-06-10.md` is the living improvement backlog** — a full June 2026 review with findings (Part 2) and a numbered to-do list (Part 3). Completed items are marked `✅ fixed/done 2026-06-10` inline WITHOUT renumbering. Check it before starting refactors: many known issues are deliberately deferred to specific numbered items, and partial fixes outside those items create churn. Update its markers when you complete an item.
- `archive/` holds historical design docs for features that have since shipped (saved assumptions, shared assumptions, editor refactor) — useful for intent/context, but the code and review doc are the source of truth where they disagree. Move plans there once their feature ships; any `*Plan.md` still at this level is presumed active.
- `*Research*.md` files document donation-data research methodology and findings.
